# 基于行为树的流程引擎组件 — 设计说明

**日期**：2026-04-11  
**目标运行时**：JDK 17+，Spring 6（通过独立 Spring 模块与父 POM 升级线对齐）  
**范围**：DAG 调度 + 重试（可关闭）；流程实例支持内存或 Redis 持久化；树形编排；v1 禁止子流程引用（纯树）；并行分支真并发，子分支失败不主动取消兄弟任务。

---

## 1. 背景与目标

在现有 `behavior-tree` 库（`BehaviorTree`、`BaseContext`、`INode`、JSON/XML 解析、Spring `EnableBehavior` 等）之上，增加一层 **流程引擎**：在保留行为树语义的前提下，提供 **可恢复实例**、**按策略重试**、**并行调度** 与 **存储抽象**，满足「只做 DAG 调度 + 重试」的编排场景，不扩展为完整 BPM/人工审批产品。

**成功标准**：

- 定义经加载与校验后可执行；v1 仅 **纯树**，无共享子流程引用。
- **展开后的执行图**（含 Parallel 汇合）保持 **有向无环**；对定义层做结构校验，并对调度视图可做防御性无环校验。
- **Retry** 支持 **按步骤类型/标签** 的默认策略（**指数退避 + 抖动**），节点可覆盖；引擎层提供 **总开关**，关闭时不执行任何重试。
- **ProcessInstanceStore**：SPI + **内存** + **Redis** 实现。
- **Parallel**：使用注入的 `Executor` **真并发**；任一侧失败 **不主动取消** 其他子任务；汇合语义与「全部子节点完成后聚合」一致（见第 4 节）。

---

## 2. 架构与边界（方案：定义 / 实例 / 引擎分层）

采用 **FlowDefinition（不可变）** + **FlowInstance（可变状态）** + **FlowEngine（驱动执行）**，在内部仍沿 **行为树语义** 遍历与执行，而非将整棵树静态编译为独立 DAG 脚本语言。

**模块划分（逻辑）**：

| 区域 | 职责 |
|------|------|
| **core / engine** | 定义加载与校验、实例生命周期、与 `BehaviorTree` 执行路径协作、重试决策（受开关约束）、Parallel 与 `Executor` 集成 |
| **store** | `ProcessInstanceStore` SPI；内存实现；Redis 实现（键空间、序列化、可选 TTL） |
| **spring6** | Spring Boot 3 / Spring 6：`Executor`、Redis、`ProcessInstanceStore` 等自动配置 |

**Retry 开关（架构级）**：

- 引擎配置项 **`retryEnabled`**（命名实现时可调整为等价布尔配置）。
- **`retryEnabled = false`**：任意节点失败 **立即** 按现有 `NodeResult`/异常路径向上传播，**不** 应用按类型/标签的重试策略（包括指数退避与抖动）。
- **`retryEnabled = true`**：在 **叶子或可重试单元** 上应用第 5 节描述的重试策略；节点级覆盖仅在开关为 true 时生效。

边界：**不** 在首版引入子流程引用；**不** 主动取消 Parallel 下未完成的兄弟任务（由业务幂等或副作用自控）。

---

## 3. 组件与数据流

**Loader**：沿用现有 JSON/XML（及扩展点）解析为内存中的 **树形定义**，产出 **FlowDefinition**（含 id、version、根节点包装等），并执行 **结构校验**（根唯一、节点类型合法、Parallel 子节点列表有效等）。

**Engine**：

- 提供实例 **启动**、**推进**；首版对外主路径为 **同步阻塞式 `run`（或等价方法）**：调用线程阻塞直至流程完成、失败或不可恢复错误。**异步 / 回调式** 执行可作为后续扩展，不纳入 v1 必交付范围。
- 执行前从 **ProcessInstanceStore** 读取快照（若存在）；在 **持久化边界**（例如节点边界、重试前后、Parallel 汇合点——实现计划细化）写回快照。
- **Parallel**：将各子分支提交至 **注入的 `Executor`**；在 **汇合点** 等待全部完成（不因某一子分支失败而取消其他任务）；聚合规则：**全部成功则成功，否则失败**（与当前 `ParallelNodeImpl` 的「全部执行后 `isSuccess` 合取」语义一致；当前实现为串行循环，引擎层升级为并发执行但 **不改变** 成功/失败判定）。

**Redis**：以 `instanceId`（及可选 `definitionId`/`version`）为主键设计 key；序列化格式在实现中选定一种 **默认**（如 JSON），并保证 **版本演进** 时可迁移或并存读取策略。

**数据流概要**：Client → Engine → [Store 读] → BT 执行路径 →（Parallel → Executor）→ [Store 写] → 结果回调/返回。

---

## 4. Parallel 汇合与失败语义

- **并发**：子节点任务提交到共享 `Executor`（线程池可配置）。
- **不取消**：任一子分支失败或抛错，**不** 向其他子任务发送取消信号。
- **汇合**：待 **所有** 已提交子任务结束（正常或异常完成）后，再计算并行节点结果；若存在失败，则该 Parallel 节点结果为失败（与现有逻辑一致）。
- **与 DAG**：Parallel 在展开图上引入 **扇出/扇入**，在 v1 纯树下仍为 DAG，无环。

---

## 5. 重试策略（开关开启时）

- **默认按步骤类型或标签** 注册 `RetryPolicy`：**最大次数**、**指数退避**、**抖动**。
- **节点可覆盖** 策略（仅当 `retryEnabled == true`）。
- **开关关闭**：本节策略 **不适用**，失败立即向上传递。

---

## 6. 错误处理

- **定义非法**：加载期失败，**不** 创建持久化实例。
- **存储失败**：明确异常类型；策略为 **fail-closed**：不向调用方谎报成功；**v1 不允许** 在 Redis/持久化失败时 **静默降级为仅内存**（若需显式「仅内存模式」须由调用方在配置层选择，而非运行时自动切换）。
- **重试耗尽**：与现有 `NodeResult.FAILURE` / 异常约定一致，并更新实例状态。

---

## 7. 测试策略

- **单元测试**：内存 `ProcessInstanceStore`；重试开关 on/off；并行汇合与不取消兄弟任务的行为；无环校验（对给定树展开图）。
- **集成测试（可选）**：Testcontainers Redis 或嵌入式 Redis，验证 Redis 实现的读写与 TTL（若采用）。

---

## 8. 与现有代码库关系

- 现有 `BehaviorTree`、`NodeResult`、`BehaviorNodeWrapper`、各类 Parser 继续作为 **定义与执行** 的基础；引擎层 **增量** 引入，避免无关大重构。
- 父工程需规划 **JDK 17** 与 **Spring 6** 与现有 **Java 8** 工件的关系（例如多模块：旧模块保持或整体升级——由实现计划与迁移策略决定，**本设计不强制单模块一次性升级**）。

---

## 9. 非目标（v1）

- 子流程引用与定义层图共享（后续版本可单独立项）。
- BPMN、人工审批、定时器作为一等公民（仅可通过业务节点间接实现则不在此承诺）。
- 强一致跨分区事务（依赖存储与业务幂等）。
